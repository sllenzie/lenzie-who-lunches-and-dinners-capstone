# Lenzie Who Lunches and Dinners

A curated, interactive restaurant directory featuring Stella Lenzie's personal recommendations.

This project combines:
- A static web app that displays restaurant cards and an interactive map
- CSV-driven content so updates are simple and data-first
- A Python geocoding script that enriches the dataset with latitude/longitude via Mapbox

## Project Goals

- Help users quickly find a place to eat from a trusted recommendation list
- Filter restaurants by cuisine, attire, and location
- Explore recommendations visually on an interactive map
- Keep the workflow reproducible so anyone can fork and adapt it to their own list

## Features

- Interactive map powered by Leaflet + Mapbox tiles
- Dynamic CSV loading with Papa Parse
- Restaurant cards with image, description, attire, cuisine, and link
- Modal detail view for each restaurant
- Multi-filter UI with smart option availability
- Responsive layout and polished visual design
- Python utility to geocode addresses into a new CSV file

## How It Works

1. The web app loads `data/lenzieswholunchanddinner_geocoded.csv`.
2. CSV rows are normalized into restaurant objects.
3. The UI renders filter options and restaurant cards.
4. Map markers are created for rows that include latitude/longitude.
5. Filter selections update both the card grid and the map in real time.

## Run the Project Locally

Because browsers block some local file access patterns, run this with a local HTTP server (not `file://`).

### Option A: Python built-in server

From the project root:

```bash
python3 -m http.server 8000
```

Then open:

```text
http://localhost:8000
```

### Option B: VS Code Live Server

If you use the Live Server extension, open `index.html` with Live Server and the app should load normally.

## Recreate This Project With Your Own Data

Use this workflow if you want to build your own version of the directory.

### 1. Prepare a source CSV

Create a CSV similar to `data/lenzieswholunchanddinner.csv` with at least:

- `Name`
- `Address`
- `Cuisine`
- `Attire`
- `Description`
- `Website`
- `Image_URL`

### 2. Set up Python dependencies

From the project root:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install pandas requests
```

### 3. Add a Mapbox token for geocoding

Open `python/geocodedlenziewholunchesanddinners.py` and update `MAPBOX_TOKEN`.

Use a token with geocoding permissions enabled.

### 4. Run the geocoding script

From the project root:

```bash
python3 python/geocodedlenziewholunchesanddinners.py
```

Expected output:
- A new/updated `data/lenzieswholunchanddinner_geocoded.csv`
- Latitude/Longitude columns populated where geocoding succeeds

### 5. Configure map tiles/style in the frontend

In `index.html`, update:

- `MAPBOX_TOKEN` (tile access token)
- `MAPBOX_STYLE_PATH` (your style id, for example `username/styleid`)

If you do not have a custom style, create one in Mapbox Studio or replace the tile source with another provider.

### 6. Run the site locally

Start a local server and open the app in your browser.

## Customization Guide

### Change filter behavior

The three active filters are defined in JavaScript as:

- cuisine
- attire
- location

To add a new filter category:
- Add a new `<select>` in the filters section of `index.html`
- Extend normalization logic to include the new key
- Include the key in `FILTER_KEYS`
- Wire events similarly to existing filters

### Change visual design

- Main styles are in `css/styles.css`
- Color system is controlled by CSS custom properties in `:root`
- Typography uses Fraunces and Space Grotesk from Google Fonts

### Change map appearance

- Marker styling is defined in `renderMap()` in `index.html`
- Popup card markup is generated by `popupHtml()` in `index.html`
- Tile style is controlled by `MAPBOX_STYLE_PATH`

## Troubleshooting

### CSV does not load

Symptoms:
- Empty page, error state, or browser console error

Fix:
- Use `http://localhost` server, not `file://`
- Confirm the CSV file path is correct
- Confirm the CSV has headers

### Map tiles do not appear

Fix:
- Verify `MAPBOX_TOKEN` is valid for tile access
- Verify `MAPBOX_STYLE_PATH` is correct
- Check token URL restrictions in your Mapbox settings

### Geocoding returns many failures or 403 errors

Fix:
- Use a token with geocoding permissions enabled
- Remove restrictive token URL/domain settings during local tests
- Retry after confirming addresses are valid and complete

## Notes on Security and Keys

- Frontend map tokens are public by design, but should still be scoped and restricted.
- Avoid committing private or secret server-side tokens.
- For production deployments, use least-privilege tokens and domain restrictions.
